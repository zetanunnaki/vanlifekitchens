# VanLifeKitchen.com

Affiliate content site for van life kitchen gear. Next.js 14 App Router, Tailwind CSS, MDX content, Kie.ai-generated imagery, Amazon Associates + Walmart affiliate links.

## Stack

- **Framework**: Next.js 14.2 (App Router, TypeScript)
- **Styling**: Tailwind CSS 3 + custom earth-tone theme
- **Content**: MDX for long-form reviews/guides/setups, JSON for structured data
- **Animation**: framer-motion
- **Icons**: lucide-react
- **Images**: Kie.ai Flux Kontext Pro (generation) → local `public/images/`
- **Deployment**: Vercel

## Local development

```bash
npm install
npm run dev
```

Dev server runs on [http://localhost:3000](http://localhost:3000).

## Content structure

```
src/content/
├── products.json           # 16 product specs + JSON-LD data
├── guides.json             # 12 guide metadata entries
├── setups.json             # 3 van build profiles
├── categories.json         # 6 product categories
├── reviews/                # Long-form MDX review bodies (16 files)
│   └── {slug}.mdx
├── guides/                 # Long-form MDX guide bodies (12 files)
│   └── {slug}.mdx
└── setups/                 # Van build MDX stories (3 files)
    └── {slug}.mdx
```

Every MDX file is 1,500+ words with SEO-structured H2 sections, internal links, and an FAQ section.

## Image generation

All product, guide, setup, category, hero, and about images are generated by [Kie.ai](https://kie.ai) using the Flux Kontext Pro model and saved locally under `public/images/`.

To regenerate images (idempotent — skips existing files):

```bash
npx tsx scripts/generate-images.ts                      # Generate everything missing
npx tsx scripts/generate-images.ts --only=reviews       # Only product images
npx tsx scripts/generate-images.ts --only=guides        # Only guide hero images
npx tsx scripts/generate-images.ts --only=setups        # Only setup images
npx tsx scripts/generate-images.ts --only=categories    # Only category tiles
npx tsx scripts/generate-images.ts --only=hero          # Only home hero
npx tsx scripts/generate-images.ts --only=hubs          # Only hub/404/og hero images
npx tsx scripts/generate-images.ts --force              # Regenerate everything
```

Requires `KIE_API_KEY` in `.env.local`:

```bash
KIE_API_KEY=your_key_here
```

Get a key at https://kie.ai/api-key.

### Aspect ratio gotcha

Flux Kontext Pro only accepts `1:1`, `16:9`, and `9:16`. Requests with `4:5`, `2:3`, `3:2` return 422.

## Affiliate integration

- **Amazon Associates** tag: `vanlifekitch-20` — applied automatically in [`src/lib/affiliate.ts`](src/lib/affiliate.ts)
- **Walmart Affiliate Program** — currently placeholder URLs in [`src/content/products.json`](src/content/products.json), need real listings before launch

No other retailers. Every product must have both an Amazon ASIN and a Walmart URL.

## SEO infrastructure

- **Sitemap**: [`src/app/sitemap.ts`](src/app/sitemap.ts) auto-generates from JSON data
- **Robots**: [`src/app/robots.ts`](src/app/robots.ts)
- **Per-page metadata**: `generateMetadata` on review, guide, setup, and category slug pages
- **JSON-LD schema**: Product + Review on review pages, Article on guide pages, ItemList on category pages, BreadcrumbList on all detail pages
- **OpenGraph / Twitter cards**: per-page images from the generated hero/product images, site-wide fallback at `/images/hero/og-default.jpg`

## Deployment to Vercel

1. Push the repo to GitHub.
2. In Vercel, import the repo.
3. Set the `KIE_API_KEY` environment variable if you plan to regenerate images in CI (not required for normal runtime).
4. Deploy. [`vercel.json`](vercel.json) configures cache headers (immutable 1-year on `/images/*`) and security headers.

### Custom domain

Point `vanlifekitchens.com` at Vercel's nameservers or add an A/CNAME record per the Vercel domain instructions. Update `metadataBase` in [`src/app/layout.tsx`](src/app/layout.tsx) if you change the canonical URL.

## Architecture decisions

- **Value-first editorial direction**: no conversion-maxxing, no dark patterns. Reviews include honest "when NOT to buy this" sections. Tools (budget, layout, power calculators) work without gating.
- **No newsletter**: deliberately removed. See `memory/feedback_vanlifekitchen_no_newsletter.md`.
- **JSON + MDX, no CMS**: simpler to deploy, faster to iterate, no database cost. Content edits are git commits.
- **Local images only**: Kie.ai temp URLs expire after 14 days, so the generation script downloads bytes to `public/images/` and updates JSON atomically.
